Delete Strategies
StrategyUsed OnWhy
CASCADEsale_units, product_tags, product_images, order_items, promotion_usagesChild data is meaningless without parent. Delete product → delete its sale units, images, tag assignments
RESTRICTproducts → categories, brands, unit_types. stock_movements, activity_logs → usersPrevent deleting a category that has products. Prevent deleting admin who has audit logs
SET NULLcategories.parent_idIf parent category deleted, children become top-level instead of being deleted
Soft Deleteproducts, categories, brands, sale_unitsNever hard delete business records. Orders reference them. Use deleted_at timestamp

Admin Pricing — Three Methods
Method 1: Direct Price Admin types: €180.00 → Stored as sellingPrice = 180.00 Method 2: Percentage Off Base: 10 × €20 = €200 | Admin types: 10% Calculated: €200 − 10% = €180.00 Method 3: Fixed Off (per base unit) Base: €20/pack | Admin types: €1.00 off/pack Calculated: 10 × (€20 − €1) = €190.00 ⚠ Base price change → admin prompted to recalculate. Never auto-updated.
Stock Availability
Product stock: 87 packs 1 Pack → floor(87 ÷ 1) = 87 available Box of 10 → floor(87 ÷ 10) = 8 available Carton of 50 → floor(87 ÷ 50) = 1 available Bulk of 100 → floor(87 ÷ 100) = 0 — OUT OF STOCK
Rules Every product must have ≥ 1 sale unit · Exactly one isDefault: true · sellingPrice is the truth for cart/checkout/orders · Stock deducted as qty × saleUnit.quantity base units
